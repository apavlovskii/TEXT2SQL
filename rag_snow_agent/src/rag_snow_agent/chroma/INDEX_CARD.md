## Card Types

### TableCard (`object_type: "table"`)
- **ID format:** `table:<DB>.<SCHEMA>.<TABLE>`
- **Document:** Table name, description, column list, time columns, join keys
- **Metadata:** `db_id`, `qualified_name`, `source`, `token_estimate`

### ColumnCard (`object_type: "column"`)
- **ID format:** `column:<DB>.<SCHEMA>.<TABLE>.<COLUMN>`
- **Document:** Column name, data type, nullable, description
- **Metadata:** `db_id`, `qualified_name`, `table_qualified_name`, `data_type`, `token_estimate`
- **Includes VARIANT sub-fields:** Nested keys from VARIANT/OBJECT/ARRAY columns are indexed as separate ColumnCards with `data_type: "VARIANT_FIELD"`. Example: `trafficSource:source`, `device:browser`, `hits:page`

### JoinCard (`object_type: "join"`)
- **ID format:** `join:<LEFT_TABLE>.<LEFT_COL>-><RIGHT_TABLE>.<RIGHT_COL>`
- **Document:** Join relationship description with confidence
- **Metadata:** `db_id`, `left_table`, `right_table`, `left_column`, `right_column`, `confidence`, `source`
- **Sources:** `fk` (foreign key constraint, confidence=1.0) or `heuristic_name` (column name matching, confidence=0.7)

### TraceRecord (in `trace_memory` collection)
- **ID format:** SHA-256 hash prefix of `instance_id:db_id:sql`
- **Document:** Instruction summary + plan summary
- **Metadata:** `db_id`, `instance_id`, `tables_used`, `token_estimate`
